Deploy Firestore Security Rules

The live Firestore rules on Firebase block admin writes (approve/reject). This page helps you publish the correct rules in under 60 seconds.

⚠ Why this happens: Your local firestore.rules file is correct, but Firebase only uses rules that are published via the Firebase Console or CLI. The live rules still have the old configuration.
1
Copy your Firestore Rules
Click the button to copy the exact rules needed. These allow xenolyth26@gmail.com and any user with role: admin to update beta requests.
2
Open Firebase Console → Firestore Rules
Click the button below to open the rules editor directly. Make sure you're signed in as the project owner.
Open Firebase Console ↗
3
Paste & Publish
In the Firebase Console rules editor:
  1. Select all existing text in the editor (Ctrl+A)
  2. Delete it
  3. Paste the copied rules (Ctrl+V)
  4. Click the blue Publish button
4
Verify & Return to Admin Console
After publishing, go back to admin.html and try Approve or Reject again. It should work immediately.
← Back to Admin Console
